Binding basics

Q1

1.Explain the relationship between a, b, c and d in the following code:

a <- 1:10
b <- a
c <- b
d <- 1:10
A1

a and d are vectors with the same values, but different memory locations.

show A1 
waldo::compare(
  x = lobstr::obj_addr(x = a), 
  y = lobstr::obj_addr(x = b))
✔ No differences
show A1 
waldo::compare(
  x = lobstr::obj_addr(x = a), 
  y = lobstr::obj_addr(x = c))
✔ No differences
show A1 
waldo::compare(
  x = lobstr::obj_addr(x = b), 
  y = lobstr::obj_addr(x = c))
✔ No differences
show A1 
waldo::compare(
  x = lobstr::obj_addr(x = a), 
  y = lobstr::obj_addr(x = d))
`old`: "0x7f8a31160f88"
`new`: "0x7f8a309f9a00"
Q2

The following code accesses the mean function in multiple ways. Do they all point to the same underlying function object? Verify this with lobstr::obj_addr().

mean
base::mean
get("mean")
evalq(mean)
match.fun("mean")
A2

These are all S3 generic functions from the base package, so they share the same underlying function object.

check get_info() 
get_info(mean)
## object: mean
##  fun type => S3, generic
##  class => function
##  type => closure
##  *mode => function
get_info(base::mean)
## object: base::mean
##  fun type => S3, generic
##  class => function
##  type => closure
##  *mode => function
get_info(get("mean"))
## object: get("mean")
##  fun type => S3, generic
##  class => function
##  type => closure
##  *mode => function
get_info(evalq(mean))
## object: evalq(mean)
##  fun type => S3, generic
##  class => function
##  type => closure
##  *mode => function
get_info(match.fun("mean"))
## object: match.fun("mean")
##  fun type => S3, generic
##  class => function
##  type => closure
##  *mode => function
show code 
lobstr::obj_addr(
  c(
    mean,
    base::mean,
    get("mean"),
    evalq(mean),
    match.fun("mean")
    )
  )
[1] "0x7f8a33c30fd8"
